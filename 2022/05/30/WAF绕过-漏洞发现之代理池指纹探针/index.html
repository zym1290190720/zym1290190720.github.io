3.启动项目

启动项目会可能会出现ERROR redis connection error: Client sent AUTH, but no password is set的错误，这是Redis没有配置密码的原因。

进入客户端命令行

输入CONFIG SET requirepass "123456"，配置密码为123456

可以输入AUTH 123456来验证123456是不是密码

或者直接修改redis.windows.conf中的requirepass修改密码

再启动项目

proxy_Pool项目有两种启动方式

python proxyPool.py schedule

python proxyPool.py server

第一种主要在终端进行，第二种可以在浏览器显示

这里采用第二种方式启动，可能会报错ImportError: cannot import name 'Markup' from 'jinja2'

这是jinja2版本的原因，查看依赖pip3 list

发现此时的Jinja2版本为3.1.0

卸载pip3 uninstall Flask Jinja2

安装pip3 install Flask Jinja2

再次查看版本

再次启动proxy_Pool项目

启动web服务后，默认配置下会开启http://127.0.0.1:5010的API接口服务

用法简介

apimethodDescriptionparams
/GETapi介绍None
/getGET随机获取一个代理可选参数：?type=https，过滤支持https的代理
/popGET获取并删除一个代理可选参数：?type=https，过滤支持https的代理
/allGET获取所有代理可选参数：?type=https，过滤支持https的代理
/countGET查看代理数量None
/deleteGET删除代理?proxy=host:ip

进入http://127.0.0.1:5010/all，获取所有可使用的代理ip，以及是否支持https

三.awvs 漏扫注入测试绕过-延时,白名单（安全狗）

1.用awvs进行扫描，如果有拦截的话，考虑以下方式绕过

  • 降低扫描速度

  • 设置百度爬虫的user-agent

  • 设置隧道代理（变换ip）

  • 工具指纹

    安全狗等waf会针对漏扫工具的指纹进行检测（只针对老款）

    数据包的特征（用户登录，修改cookie，修改授权等等来绕过WAF对工具指纹的检测）

  • payload的修改（举几个工具扫描注入的payload）

    and 1=1 awvs的注入方式

    or 1=1 xray注入

    xor 1=2 appscan注入

2.总结

  • 自己开发工具或者采用开源的工具（将payload的数据变异，可以结合之前SQL注入绕过安全狗的fuzz方法变异payload）；

  • 采用才出来的漏洞扫描工具，或者冷门扫描工具；

  • 如果扫描器没有控制扫描速度的功能，可以设置代理发包到burpsuite上，然后使用按键精灵控制发包速度 。
