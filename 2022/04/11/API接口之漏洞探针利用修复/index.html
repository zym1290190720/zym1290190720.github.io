二.API接口-WebServiceRES Tful API

根据应用自身的功能方向决定，安全测试目标需有 API 接口调用才能进行此类测试，主要涉及的安全问题：自身安全，配合 WEB，业务逻辑等，其中产生的危害可大可小，属于应用 API 接口网络服务测试面，一般也是在存在接口调用的情况下的测试方法。

1.rpc（远程过程调用）

2.webservice（漏洞主要存在WSDL（XML的一种））

3.RESTful API

4.MVC/MVP/MVVM

MVC 将整个应用分成 Model、View和Controller三个部分，而这些组成部分其实也有着几乎相同的职责。

视 图：管理作为位图展示到屏幕上的图形和文字输出；

控制器：翻译用户的输入并依照用户的输入操作模型和视图；

模 型：管理应用的行为和数据，响应数据请求（经常来自视图）和更新状态的指令（经常来自控制器）。

注：tomcat除了可能存在弱口令漏洞外，还有其他的漏洞可能存在，比如CVE-2020-1938（该漏洞可以读取Tomcat目录中webapp下的任意文件）。此处通过nmap扫描得知，与该漏洞相关的8009端口没有开启（即AJP协议没有开启），因此不存在该漏洞。
